TITLE: Todoist API v1 - Share Project Command
DESCRIPTION: Demonstrates how to share a project with another user using the Todoist API's sync endpoint. Includes example request and response, and details command arguments.

SOURCE: https://developer.todoist.com/api/v1/index

LANGUAGE: APIDOC
CODE:
```
Share a project with another user.

When sharing a teams project:
Workspace projects with `is_invite_only` set to true can only be shared by workspace admins, or by project members with `ADMIN` or `CREATOR` role. Other users will get a “forbidden” error. The role for the new collaborator cannot be greater than the role of the person sharing the project.

Command arguments:
  project_id (String, Required): The project to be shared.
  email (String, Required): The user email with whom to share the project.
  role (String, Optional): The role of the new collaborator in the workspace project. If missing, the workspace `collaborator_role_default` will be used. Only used for teams.
```

LANGUAGE: bash
CODE:
```
$ curl https://api.todoist.com/api/v1/sync \
    -H "Authorization: Bearer 0123456789abcdef0123456789abcdef01234567" \
    -d commands='[
    {
        "type": "share_project",
        "temp_id": "854be9cd-965f-4ddd-a07e-6a1d4a6e6f7a",
        "uuid": "fe6637e3-03ce-4236-a202-8b28de2c8372",
        "args": {

DESCRIPTION: Provides examples of JSON payloads for various live notification types in the Todoist API. These notifications inform clients about real-time changes within a Todoist project or account.

SOURCE: https://developer.todoist.com/api/v1/index

LANGUAGE: APIDOC
CODE:
```
Todoist API Live Notifications:

This section provides examples of the JSON structure for various live notification types received from the Todoist API. These notifications are used to inform clients about real-time events and changes within their Todoist account.

**Notification Types and Structures:**

1.  **share_invitation_sent**
    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `invitation_id`: ID of the invitation.
    *   `invitation_secret`: Secret key for the invitation.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "share_invitation_sent".
    *   `seq_no`: Sequence number for ordering notifications.
    *   `state`: Current state of the invitation (e.g., "accepted").

2.  **share_invitation_accepted**
    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `invitation_id`: ID of the invitation.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "share_invitation_accepted".
    *   `project_id`: ID of the project the invitation was for.
    *   `seq_no`: Sequence number for ordering notifications.

3.  **share_invitation_rejected**
    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `invitation_id`: ID of the invitation.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "share_invitation_rejected".
    *   `project_id`: ID of the project the invitation was for.
    *   `reject_email`: Email address that rejected the invitation.
    *   `seq_no`: Sequence number for ordering notifications.

4.  **user_left_project**
    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "user_left_project".
    *   `project_id`: ID of the project the user left.
    *   `seq_no`: Sequence number for ordering notifications.

    *   `seq_no`: Sequence number for ordering notifications.

    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `item_content`: Content of the assigned item.
    *   `item_id`: ID of the assigned item.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "item_assigned".
    *   `project_id`: ID of the project the item belongs to.
    *   `responsible_uid`: User ID of the person the item is assigned to.
    *   `seq_no`: Sequence number for ordering notifications.

7.  **item_completed**
    *   `assigned_by_uid`: User ID of the person who assigned the item.
    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `item_content`: Content of the completed item.
    *   `item_id`: ID of the completed item.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "item_completed".
    *   `project_id`: ID of the project the item belongs to.
    *   `responsible_uid`: User ID of the person the item was assigned to.
    *   `seq_no`: Sequence number for ordering notifications.

8.  **item_uncompleted**
    *   `assigned_by_uid`: User ID of the person who assigned the item.
    *   `created_at`: Timestamp of notification creation.
    *   `is_unread`: Boolean indicating if the notification is unread.
    *   `from_uid`: User ID of the sender.
    *   `id`: Unique ID of the notification.
    *   `item_id`: ID of the uncompleted item.
    *   `item_content`: Content of the uncompleted item.
    *   `notification_key`: A key associated with the notification.
    *   `notification_type`: Set to "item_uncompleted".
    *   `project_id`: ID of the project the item belongs to.
